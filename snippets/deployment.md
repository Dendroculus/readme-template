# Deployment

```markdown
## 🚀 Deployment

1. Configure production secrets through the platform.
2. Use production database and storage URLs.
3. Set allowed origins and proxy behavior.
4. Run migrations before serving traffic.
5. Install required browser or system runtimes.
6. Start workers or schedulers only when the architecture requires them.
7. Configure health checks and graceful shutdown.
8. Verify cleanup, retention, and backup behavior.
9. Monitor errors, latency, queue depth, and storage growth.
```

Avoid platform-specific claims unless the repository includes that deployment configuration.
